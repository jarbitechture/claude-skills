# Phase 2: Template Strategy Selection

## Purpose

Present field mapping recommendations and allow user confirmation before template generation.

## Process

1. Review field inventory from Phase 1
2. Propose template pattern based on data characteristics
3. Recommend field mappings (frontmatter, body, tags, links)
4. Confirm note naming field
5. Allow user customization

## Outputs

- Selected template pattern
- Confirmed field mappings
- Note naming strategy
- Tag conventions
- Link relationships
